⚠ The Quantum Threat

Shor's algorithm can factor large integers and solve discrete logarithm problems in polynomial time on a cryptographically relevant quantum computer (CRQC). This breaks RSA, ECDSA, ECDH, and DH — the foundations of current internet security.

✓ The PQC Response

Post-quantum cryptography uses mathematical problems believed hard for both classical and quantum computers. NIST standardised three algorithms in 2024 based on lattice problems and hash functions — immune to Shor's and Grover's algorithms.

Hard Problem — Learning With Errors

Given a matrix A and vector b = As + e (mod q), find the secret s — where e is a small-magnitude "error". With random A and appropriate parameters, this is computationally indistinguishable from a random vector even for quantum computers.

FIPS 204 · 2024

ML-DSA

Module Lattice-based Digital Signature Algorithm (Dilithium)

Based on: MSIS + MLWE
Formerly: CRYSTALS-Dilithium
Fiat-Shamir: with aborts

Hard Problem — MSIS

Module Short Integer Solution: find a short vector z such that Az = 0 (mod q) in a lattice module. Combined with MLWE for the "Fiat-Shamir with aborts" construction that avoids leaking secret information through rejection sampling.

Fiat-Shamir with Aborts

Unlike classical Schnorr signatures, the response vector may leak secret info if too large. ML-DSA uses rejection sampling: if the response exceeds a bound, the signing is aborted and restarted. This ensures the distribution is independent of the secret key.

FIPS 205 · 2024

SLH-DSA

Stateless Hash-Based Digital Signature Algorithm (SPHINCS+)

Based on: Hash functions only
Formerly: SPHINCS+
Trees: Hypertree XMSS

Why Hash-Based?

SLH-DSA's security relies only on the collision resistance and one-wayness of SHA-256 or SHAKE. Unlike lattice schemes, its security assumptions are decades-old and extremely well-studied. It is the conservative "insurance policy" in the NIST portfolio — if lattice assumptions are unexpectedly broken, SLH-DSA remains secure.

Migration Strategy

  • Hybrid mode: combine classical + PQC temporarily
  • Harvest now, decrypt later — start encrypting data today
  • TLS 1.3 + ML-KEM already supported in Chrome/Firefox (X25519MLKEM768)
  • PKI migration: leaf certs first, then intermediates, then roots
  • KERI/ACDC: self-certifying identifiers can rotate to PQC keys

NIST Security Levels

  • Level 1 — AES-128 equivalent (Grover attack)
  • Level 2 — SHA-256 collision equivalent
  • Level 3 — AES-192 equivalent
  • Level 4 — SHA-384 collision equivalent
  • Level 5 — AES-256 equivalent (highest)
